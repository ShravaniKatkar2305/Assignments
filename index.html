MySQL Notes — Quick Reference

Concise, copy-paste examples and a cheat sheet for everyday SQL tasks

Overview

MySQL is a relational database management system (RDBMS). Use SQL (Structured Query Language) to create, read, update and delete (CRUD) data. These notes cover essentials: installation, common commands, datatypes, joins, indexing and handy examples.

Installation & Connect

  • Install: Use package manager (apt/yum/brew) or download from MySQL site.
  • Start server: sudo service mysql start or systemctl.
  • Connect from terminal: mysql -u root -p
  • Create DB & User: 
    CREATE DATABASE school;
CREATE USER 'anna'@'localhost' IDENTIFIED BY 'StrongPass123';
GRANT ALL PRIVILEGES ON school.* TO 'anna'@'localhost';
FLUSH PRIVILEGES;

Basic SQL Commands (CRUD)

Create table

CREATE TABLE students (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  roll_no VARCHAR(20) UNIQUE,
  dob DATE,
  score DECIMAL(5,2),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

Insert

INSERT INTO students (name, roll_no, dob, score)
VALUES ('Riya Sharma','RN001','2006-08-15', 78.50);

Select (read)

SELECT * FROM students;
SELECT id, name FROM students WHERE score >= 75 ORDER BY score DESC LIMIT 10;

Update

UPDATE students
SET score = 82.00
WHERE roll_no = 'RN001';

Delete

DELETE FROM students WHERE id = 123;

Joins — Quick Examples

Assume tables students(id,name,roll_no) and marks(id,student_id,subject,score)

INNER JOIN

SELECT s.name, m.subject, m.score
FROM students s
INNER JOIN marks m ON s.id = m.student_id;

LEFT JOIN

SELECT s.name, m.subject, m.score
FROM students s
LEFT JOIN marks m ON s.id = m.student_id;

RIGHT JOIN

SELECT s.name, m.subject, m.score
FROM students s
RIGHT JOIN marks m ON s.id = m.student_id;

CROSS JOIN

SELECT s.name, c.course_name
FROM students s
CROSS JOIN courses c; -- Cartesian product

Indexes & Performance

  • Create index: CREATE INDEX idx_roll ON students(roll_no);
  • Indexes speed up reads but slow inserts/updates. Index columns used in WHERE, JOIN, ORDER BY.
  • Use EXPLAIN SELECT ... to see query plan.

Transactions

Use transactions for multiple dependent changes:

START TRANSACTION;
UPDATE accounts SET balance = balance - 100 WHERE id = 1;
UPDATE accounts SET balance = balance + 100 WHERE id = 2;
COMMIT; -- or ROLLBACK;

Backup / Restore

  • Dump (export): mysqldump -u user -p dbname > backup.sql
  • Restore: mysql -u user -p dbname < backup.sql

Handy Tips & Best Practices

  • Always use parameterized queries/prepared statements to avoid SQL injection.
  • Normalize data but denormalize when necessary for performance.
  • Use clear naming conventions (snake_case or camelCase consistently).
  • Keep small, focused transactions and avoid long-running transactions.
  • Regularly run OPTIMIZE TABLE on heavily updated tables if necessary.

Small Example: Create & Query

-- create database and tables
CREATE DATABASE bookshop;
USE bookshop;

CREATE TABLE authors (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(150) NOT NULL
);

CREATE TABLE books (
  id INT AUTO_INCREMENT PRIMARY KEY,
  author_id INT,
  title VARCHAR(255),
  price DECIMAL(8,2),
  FOREIGN KEY (author_id) REFERENCES authors(id)
);

-- insert & join
INSERT INTO authors (name) VALUES ('M. Patel');
INSERT INTO books (author_id, title, price) VALUES (1,'SQL Basics', 299.00);

SELECT b.title, a.name, b.price
FROM books b
JOIN authors a ON b.author_id = a.id;
