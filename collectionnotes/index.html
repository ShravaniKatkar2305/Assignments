What is collection?

The Collection is a ready-to-use implementation of data structure provided by Java.
Collection is a set of objects stored in a single variable.
The Object class is the parent of every class.
Primitive data types are not supported in collection.

Key Features:

  • Root interface of Java Collections Framework.
  • Provides common methods like add(), remove(), size(), clear(), isEmpty().
  • Supports iteration using Iterator.
  • Implemented by various subinterfaces like List, Set, and Queue.
  • Does not allow primitive types directly; use wrapper classes (Integer, Double, etc.).

List in Java (Collection Framework)

What is List?

List is an interface in Java that stores ordered data and allows duplicate values. Elements are accessed using index numbers (0,1,2...).

Types of List

1. ArrayList

  • Fast for reading/searching
  • Slow for inserting/deleting in the middle
  • Used most commonly
ArrayList<String> list = new ArrayList<>();
list.add("Apple");
list.add("Banana");
list.add("Mango");

2. LinkedList

  • Fast for insertion and deletion
  • Slower for searching
  • Uses nodes linked to each other
LinkedList<String> list = new LinkedList<>();
list.add("Monday");
list.add("Tuesday");
list.add("Wednesday");

3. Vector

  • Similar to ArrayList
  • Thread-safe (synchronized)
  • Slower compared to ArrayList
Vector<Integer> v = new Vector<>();
v.add(10);
v.add(20);
v.add(30);

4. Stack

  • Follows LIFO: Last In First Out
  • Push to add, Pop to remove
Stack<String> s = new Stack<>();
s.push("A");
s.push("B");
s.push("C");
s.pop();   // removes C

Set in Java (Simple Explanation)

What is Set?

A Set is a collection that does not allow duplicate values. It is used when you want only unique items.

Types of Set

1. HashSet

• Stores elements in random order
• Fastest
• No duplicates

HashSet set = new HashSet<>();
set.add("Apple");
set.add("Mango");
set.add("Apple"); // duplicate ignored
System.out.println(set);

2. LinkedHashSet

• Maintains insertion order
• No duplicates

LinkedHashSet set = new LinkedHashSet<>();
set.add("Dog");
set.add("Cat");
set.add("Cow");
System.out.println(set);

3. TreeSet

• Stores elements in sorted order
• No duplicates

TreeSet nums = new TreeSet<>();
nums.add(40);
nums.add(10);
nums.add(20);
System.out.println(nums);

Map in Java (Simple Notes)

What is Map?

Map is used to store data in key-value pairs. Each key is unique but values can be duplicate.

1. HashMap

✔ Stores key-value pairs

✔ No order maintained

✔ Allows one null key

HashMap map = new HashMap<>();
map.put(101, "Riya");
map.put(102, "Om");

2. LinkedHashMap

✔ Maintains insertion order

✔ Slightly slower than HashMap

LinkedHashMap map = new LinkedHashMap<>();
map.put(1, "Apple");
map.put(2, "Banana");

3. TreeMap

✔ Stores keys in sorted order

✔ Does not allow null key

TreeMap map = new TreeMap<>();
map.put(10, "Blue");
map.put(20, "Green");

4. Hashtable

✔ Thread-safe

✔ Does not allow null key/value

Hashtable map = new Hashtable<>();
map.put(1, "Java");

5. SortedMap

✔ Parent interface of TreeMap

6. NavigableMap

✔ Child interface of SortedMap

✔ Provides navigation methods like lowerKey(), higherKey()
